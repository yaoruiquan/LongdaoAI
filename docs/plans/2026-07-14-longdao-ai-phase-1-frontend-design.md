# 龙道 AI 第一阶段前端改版设计

**日期：** 2026-07-14  
**状态：** 已确认  
**项目：** Sub2API Token 中转站改造  
**本阶段范围：** 前端品牌、信息架构与业务界面改版，不接入真实支付、不修改计费后端

## 1. 目标

将现有 Sub2API 前端改造成面向开发者与企业客户的 **龙道 AI 企业级 AI API 服务平台**。在保持现有 Vue 3、Pinia、Vue Router、Tailwind、暗色模式、中英文切换和既有 API 行为兼容的前提下，完成第一阶段可演示、可继续开发的前端基线。

本阶段的核心结果：

- 公开页面具有完整的龙道 AI 品牌识别和企业级产品表达。
- 登录、注册和控制台形成一致的视觉系统。
- 用户可以清楚理解 API Key、套餐、余额、消费和订单之间的关系。
- 管理员菜单按照运营、用户、资源、财务、风控重新组织。
- 尚无后端支持的页面使用真实、诚实的建设中状态，不伪造支付成功、余额变化或订单数据。

## 2. 品牌与视觉方向

### 2.1 品牌名称

- 对外产品名：**龙道 AI**
- 公司及版权名：**龙道集团**
- 产品定位：**企业级 AI API 服务平台**
- 首页主标题：**稳定连接全球 AI 能力**
- 首页副标题：**龙道集团为开发者与企业提供安全、稳定、透明计费的 AI API 接入服务。**

### 2.2 Logo 与颜色

- 原始 Logo：`/Users/yao/Pictures/favicon.ico`
- 品牌主色：`#EB3F00`
- 浅色主题：白色、暖灰、深灰为主体，品牌红用于关键按钮、强调线、状态和数据高亮。
- 暗色主题：接近黑色的深灰背景，品牌红用于高对比强调，避免大面积高饱和红造成疲劳。
- 避免通用紫色 AI 渐变；采用克制的红色光晕、细网格、数据线和半透明层次。

### 2.3 视觉特征

- 企业科技风，强调可靠、清晰和可审计感。
- 页面使用明确的信息层级、宽松留白、稳定的卡片栅格和少量有目的的动效。
- 首页突出“连接、路由、计量、保护”四类平台能力。
- 登录注册采用桌面端左右分栏：左侧品牌价值与服务指标，右侧表单；移动端收敛为单栏。
- 控制台优先展示余额、套餐额度、今日消耗、API Key 状态和接入引导。

## 3. 信息架构

### 3.1 公共首页

首页包含：

1. 品牌导航：Logo、龙道 AI、产品能力、支持模型、计费优势、登录、注册。
2. Hero：主标题、副标题、进入控制台和查看接入方式 CTA。
3. 平台数据概览：使用可验证或明确为能力指标的文案，避免伪造实时业务数据。
4. 产品能力：统一接入、稳定路由、透明计量、安全控制。
5. 支持模型：以兼容能力和模型类别展示，不承诺未验证的供应商可用性。
6. 集成步骤：创建 API Key、替换 Base URL、发起请求。
7. 计费优势：清晰展示人民币结算、美元源成本参考、套餐优先、余额兜底的目标规则。
8. 服务保障：稳定性、权限隔离、日志审计、技术支持。
9. 页尾 CTA 与龙道集团版权信息。

### 3.2 用户控制台导航

导航分组为：

- **工作台**
  - 工作台

  - API 密钥
  - 用量与消费
- **资金与套餐**
  - 套餐中心
  - 余额充值
  - 订单记录

兼容已有简单模式、Sora 客户端开关、自定义菜单和管理员个人菜单。

管理员菜单按业务域分组：

- **运营总览**：管理概览、运维监控、公告管理
- **用户与权益**：用户管理、分组管理、套餐管理、兑换码、优惠码

- **财务**：用量记录、支付订单、资金流水

已有功能继续连接原页面；支付订单、资金流水和风险事件使用“功能建设中”页面，后续替换为真实后端。

## 4. 页面设计

### 4.1 登录与注册

- 使用统一 `AuthLayout` 品牌分栏。
- 保留现有邮箱/用户名、密码、TOTP、LinuxDo OAuth、找回密码等逻辑。
- 注册继续兼容后端公开设置控制的邮箱验证码、邀请码、推广码和 Turnstile。
- 不通过视觉重构改变验证顺序、错误处理或请求格式。
- 表单错误、提交中、禁用和网络异常状态必须清楚可见。

### 4.2 工作台

- 顶部欢迎区展示平台定位和主要操作。
- 数据卡包含：可用余额、套餐剩余额度、今日消费、API Key 数量或状态。
- 显示“套餐额度优先消耗，额度用尽后使用余额”的规则说明。
- 数据只来自现有用户与用量 API；缺失字段显示占位说明而不是构造数值。
- 增加快速接入、创建 Key、购买套餐、余额充值入口。

### 4.3 API 密钥与使用说明

- 保留现有 Key 创建、编辑、删除、复制等功能。
- 在页面加入接入指南：Base URL、Bearer 鉴权、OpenAI SDK、Claude SDK、cURL。
- 代码示例应可复制，并明确不要在浏览器或公开仓库暴露密钥。
- Base URL 优先使用当前站点来源或后端已有配置，不硬编码生产域名。

### 4.4 套餐中心

- 继续读取现有订阅产品和用户订阅 API。
- 套餐卡展示名称、价格、额度、周期、倍率、适用模型和权益。
- 购买按钮不调用真实支付；按钮和说明明确标记“支付系统接入中”。
- 可保留已有管理员发放或兼容逻辑，但面向用户的支付动作不得伪造成功。

### 4.5 充值、余额、消费与订单

- 余额充值页包含余额概览、充值金额选择、支付方式占位、充值说明。
- 所有支付按钮禁用或进入说明状态，并明确“支付系统尚未接入，暂不可在线充值”。
- 订单页展示未来订单字段和空状态，不生成虚假订单。
- 用量与消费页以人民币显示用户计费金额，并保留美元源成本参考。
- 统一货币格式：人民币 `¥` 为主，美元 `$` 为源成本或参考成本。

### 4.6 管理后台占位页

- 支付订单、资金流水、风险事件页面使用统一建设中组件。
- 页面展示预计能力、当前状态和后续依赖，不出现可误操作的真实按钮。
- 后续接入 API 时保留路由和信息架构，替换内容组件即可。

## 5. 技术实现

### 5.1 架构原则

- 保持现有 Vue 3 Composition API、TypeScript、Pinia、Vue Router、vue-i18n 和 Tailwind CSS。
- 不为纯视觉改版新增大型依赖。
- 品牌令牌集中在 `tailwind.config.js` 和全局样式中。
- 建立可复用的品牌标识、代码示例、建设中状态和资金摘要组件。
- 所有新增文本同时提供中文和英文翻译。

### 5.2 资源处理

- 将用户提供的 ICO 复制为 `frontend/public/favicon.ico`。
- 生成透明 PNG 版本供页面 Logo 使用，例如 `frontend/public/longdao-logo.png`。
- `index.html` 默认标题和 favicon 改为龙道 AI。
- 后端公开设置仍可覆盖站点名称和 Logo；缺省值使用龙道 AI 品牌。

### 5.3 数据和行为边界

- 不更改现有 API 请求路径和载荷。
- 不实现支付后端、资金入账、真实订单或计费规则切换。
- 不通过前端写入虚假余额或订阅。
- 缺少数据时提供加载、空状态、错误状态和重试入口。
- 后端实际计费规则在后续阶段实现；本阶段仅展示已确认的目标规则说明。

## 6. 响应式、主题与可访问性

- 桌面端保持侧栏布局，移动端保持抽屉式导航。
- 登录注册在小屏改为单栏，隐藏非必要装饰但保留品牌识别。
- 表格在移动端允许横向滚动，并优先保留关键字段。
- 所有关键颜色满足可读性要求；焦点状态、禁用状态和错误状态不能只依赖颜色表达。
- 暗色主题与浅色主题均需要检查 Logo、边框、代码块和红色按钮对比度。
- 保留中英文切换，并验证长英文文本不会破坏布局。

## 7. 验收标准

### 7.1 功能

- 首页、登录、注册、控制台、API Key、用量、套餐、充值、订单、账户设置和管理员菜单可正常访问。
- 现有登录、注册、Key、用量、订阅和管理员功能没有因视觉改版改变请求行为。
- 支付相关页面明确显示未接入状态，不产生成功提示、余额变化或订单。
- 用户与管理员导航分组正确，简单模式与权限守卫仍有效。

### 7.2 视觉

- 全站默认品牌为龙道 AI，版权为龙道集团。
- 主色为 `#EB3F00` 红白体系，暗色模式使用深灰黑与品牌红。
- 首页和认证页在桌面、平板、手机尺寸下无明显溢出或遮挡。
- 中文和英文、浅色和暗色模式均保持可用。

### 7.3 工程

- ESLint 通过。
- TypeScript 类型检查通过。
- 现有测试及新增测试通过。
- Vite 生产构建通过。
- Docker 开发镜像构建并启动成功。
- `http://127.0.0.1:3000` 与 `http://127.0.0.1:8080` 页面和健康检查可访问。
